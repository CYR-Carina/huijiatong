# 🧠 惠家通 — 乡村家庭智能服务 Agent

## 2026 年度 "火山杯" Agent 创新大赛（深信大国赛遴选赛）项目介绍

---

## 一、项目概述

### 1.1 项目名称

**惠家通** — 乡村家庭智能服务 Agent

### 1.2 选题方向

校园服务 / 产业应用

### 1.3 一句话定位

一个 Agent，服务整个农村家庭——学生问资助，老人防诈骗，全家享便利。

### 1.4 项目背景

本项目灵感来源于深圳信息职业技术大学 2026 年暑期三下乡社会实践活动——"铸信助学·青春护航——资助政策宣讲与关爱帮扶行动"。

在为期 5 天的线下服务中，团队通过入户走访、政策宣讲会、集市摆摊、银龄关怀主题活动等方式，深入服务了广东省新田镇的家庭经济困难学生、留守儿童及独居老人。在此过程中，我们发现了两大突出问题：

- **资助政策信息不对称**：超过 70% 的困难家庭对国家资助政策（助学贷款、助学金、奖学金等）知之甚少，部分家庭甚至因"不知道"而错过了申请机会；
- **银龄群体信息防护薄弱**：留守老人是诈骗电话的高发群体，同时缺乏健康科普和养老政策信息的获取渠道。

基于这一真实痛点，我们设计了 **「惠家通」**——一个面向农村家庭的智能服务 Agent，用语音交互的方式，让不同年龄段的家庭成员都能便捷地获取所需信息。

## 二、目标用户与使用场景

### 2.1 目标用户

| 用户画像 | 身份特征 | 核心需求 |
|:---------|:---------|:---------|
| 农村高中生/大学生家长 | 40-60 岁，文化程度不高，不擅用手机 | 了解资助政策、申请流程 |
| 农村在校学生（初/高/大学） | 15-25 岁，有一定上网能力 | 查询可申请的资助项目、办理流程 |
| 农村留守老人 | 60 岁以上，视力/听力下降，不识字 | 识别诈骗电话、健康科普、养老金查询 |
| 基层学校资助老师/村干部 | 负责政策宣传，但信息更新不及时 | 快速查询政策细节、辅助走访 |

### 2.2 使用场景

- **家庭场景**：家长或学生在家里通过语音提问，Agent 即时回答资助政策问题；
- **入户走访场景**：三下乡志愿者或村干部在走访时，用 Agent 辅助解答政策疑问；
- **银龄关怀场景**：老人独自在家，遇到可疑电话或健康问题时，通过语音向 Agent 求助；
- **集市/社区活动场景**：在集市摆摊或社区活动中，设置 Agent 互动体验点，吸引群众了解政策。

## 三、核心功能

### 3.1 功能总览

```
🧠 惠家通
│
├── 📚 模式一：资助政策通（学生/家长模式）
│   ├── 政策问答：助学贷款、助学金、奖学金、绿色通道、勤工助学
│   ├── 条件自测：输入家庭情况 → 智能推荐匹配资助项目
│   └── 申请流程指引：分步操作指南 + 材料清单 + 时间节点
│
├── 👴 模式二：银龄小保（老人模式）
│   ├── 诈骗识别劝阻：可疑电话分析、常见骗局科普
│   ├── 健康科普：慢性病注意事项、日常养生建议
│   └── 养老政策查询：养老金、医保报销、高龄补贴、社保卡
│
└── 🔄 通用功能
    ├── 全语音交互（语音输入 → 语音回答，全程闭环）
    ├── 大字版 UI（按钮 ≥ 48px，字体 ≥ 20px）
    ├── 模式自动切换（关键词识别 + LLM 意图分类）
    ├── 安全边界（防诈骗警示、医疗免责、政策溯源）
    └── 结果可转发（一键分享给子女/村干部）
```

### 3.2 功能详细说明

#### 3.2.1 政策问答

用户通过语音或文字输入关于资助政策的问题，Agent 基于知识库进行检索并生成回答。支持多轮对话，可在同一话题下深入追问。

**示例对话：**

> 用户：「我家孩子读高三，能申请什么资助？」
>
> Agent：「您好！根据您的情况，孩子高三毕业后可以申请以下资助：
>
> 1️⃣ 生源地信用助学贷款 — 最高 16000 元/年，读书期间国家贴息
> 2️⃣ 国家助学金 — 入学后申请，平均 3300 元/年
> 3️⃣ 绿色通道 — 没带学费也能先报到
>
> 需要我详细介绍哪一个？」
>
> 用户：「助学贷款怎么申请？」
>
> Agent：「生源地助学贷款申请流程（4 步）：
>
> ① 每年 7-9 月在户籍所在地县教育局办理
> ② 需要材料：录取通知书、身份证、户口本、共同借款人身份证
> ③ 共同借款人要求：父母或其他法定监护人，年龄不超过 60 岁
> ④ 贷款金额：本专科生最高 16000 元/年
>
> 需要我列出具体需要带什么材料吗？」

#### 3.2.2 条件自测

用户输入家庭经济情况，Agent 自动匹配可申请的资助项目，并按优先级排序推荐。

**示例对话：**

> 用户：「我家年收入 3 万，两个孩子上学，一个高中一个大学」
>
> Agent：「根据您的情况，我为您推荐以下资助项目（按优先级排序）：
>
> 1️⃣ 生源地助学贷款 — 两个孩子都可申请，最高 16000 元/人/年
> 2️⃣ 国家助学金 — 平均 3300 元/人/年
> 3️⃣ 学校困难补助 — 入学后向辅导员申请
> 4️⃣ 勤工助学 — 校内工作，每月约 300-800 元
>
> 需要我帮您生成一份详细的申请时间表和材料清单吗？」

#### 3.2.3 诈骗识别劝阻

当用户描述可疑电话或信息时，Agent 即时判断是否为诈骗，并给出劝阻建议和官方联系方式。

**示例对话：**

> 用户：「有人打电话说我中奖了，让我交 500 元手续费才能领奖」
>
> Agent：「⚠️ 重要提醒：这是诈骗！请立即挂断电话！
>
> ✅ 记住：中奖不需要提前交任何费用
> ✅ 不要提供银行卡号、密码、验证码
> ✅ 如有疑问，请联系村干部或拨打 110 咨询」

#### 3.2.4 健康科普

针对老年人常见慢性病和健康问题，Agent 提供通俗易懂的科普知识，并引导及时就医。

**示例对话：**

> 用户：「我有高血压，平时要注意什么？」
>
> Agent：「给您一些高血压日常注意事项：
>
> 🥗 饮食方面：少吃盐，每天不超过 5 克；多吃蔬菜水果
> 🏃 生活方面：每天散步 30 分钟；保持心情愉快
> 💊 用药方面：按时吃药，不要自己停药
>
> 📌 提醒：我是科普助手，不能代替医生，不舒服请及时就医。」

#### 3.2.5 模式自动切换

Agent 支持通过关键词识别和 LLM 意图分类，自动判断用户当前的诉求属于哪个模式，无需手动切换。

**示例切换：**

> 用户（在银龄模式下）：「那我想问一下，我家孩子助学贷款怎么办？」
>
> Agent：「已自动切换到资助政策模式！生源地助学贷款申请流程如下：……」

## 四、技术方案

### 4.1 技术架构

```
┌─────────────────────────────────────────────────────┐
│                    用户交互层                          │
│  🎤 语音输入 → [火山引擎 ASR] → 文字                 │
│  📱 文字输入（备用）                                  │
└──────────────────────┬──────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────┐
│               Trae 开发平台（火山引擎）                 │
│  ┌──────────────────────────────────────────────┐   │
│  │  意图识别与路由                                  │   │
│  │  用户输入 → 意图分类器 → 判断：                  │   │
│  │  ├── 资助政策类 → 路由到资助知识库               │   │
│  │  ├── 银龄关怀类 → 路由到银龄知识库               │   │
│  │  └── 模糊/跨类 → 追问确认身份                    │   │
│  └────────────────┬─────────────────────────────┘   │
│  ┌────────────────▼─────────────────────────────┐   │
│  │  双知识库（RAG 检索增强生成）                    │   │
│  │  📚 资助政策库          📚 银龄关怀库           │   │
│  │  └── 用户提问 → 向量化 → 检索 → 生成回答        │   │
│  └────────────────┬─────────────────────────────┘   │
│  ┌────────────────▼─────────────────────────────┐   │
│  │  Agent 核心（豆包大模型 + 双 Prompt 模板）       │   │
│  └────────────────┬─────────────────────────────┘   │
│  ┌────────────────▼─────────────────────────────┐   │
│  │  安全过滤层 + 输出（文字 + TTS 语音）            │   │
│  └─────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────┘
```

### 4.2 技术选型

| 层级 | 技术组件 | 说明 |
|:-----|:---------|:-----|
| 开发平台 | **火山引擎 Trae** | 自然语言驱动 AI 开发，零基础可用 |
| 大模型 | **豆包大模型**（火山方舟） | 支持 RAG、工具调用、多轮对话 |
| 语音识别 | **火山引擎 ASR** | 中文普通话语音转文字 |
| 语音合成 | **火山引擎 TTS** | 多音色可选，支持语速调节 |
| 知识库 | **Trae 知识库** | 双知识库架构，自动切片+向量化 |
| 意图识别 | 关键词规则 + **豆包大模型**分类 | 双层分类，准确率高 |

| | 广东省教育厅 | 广东省学生资助政策 |
| | 深圳信息职业技术大学 | 校内资助手册、绿色通道流程 |
| | 三下乡走访 | 真实 FAQ 清单 |
| 银龄关怀库 | 公开防诈骗指南 | 常见诈骗类型与识别方法 |
| | 健康科普资料 | 慢性病管理、日常养生知识 |
| | 人社/医保部门公开文件 | 养老金、医保报销、社保卡政策 |
| | 三下乡走访 | 农村老人常见问题 FAQ |

### 4.4 安全边界设计

| 场景 | 处理方式 |
|:-----|:---------|
| 涉及具体金额 | 标注"以当年官方政策为准" |
| 医疗/健康问题 | 提供科普知识，标注"我是科普助手，不能代替医生，不舒服请及时就医" |
| 诈骗类问题 | 红色警示标志 + 劝阻建议 + 官方热线 |
| 政策未覆盖 | 引导联系学校资助中心或拨打 010-66097980 |
| 超出知识库范围 | 明确告知"这个问题我暂时无法回答，建议您咨询相关部门" |

## 五、创新性与特色

### 5.1 创新点

| 创新点 | 说明 |
|:-------|:-----|
| **家庭化服务视角** | 区别于面向单一用户的 Agent，惠家通以"农村家庭"为单位，一个 Agent 覆盖学生和老人两类人群的需求 |
| **双模式融合架构** | 资助政策 + 银龄关怀两套知识库和 Prompt 体系，通过意图识别实现无缝切换 |
| **适老化语音交互** | 全语音闭环 + 大字版 UI + 慢速语速 + 一键重听，专为农村老年群体设计 |
| **真实场景驱动** | 项目灵感来源于三下乡实地走访，功能设计基于真实需求调研 |

### 5.2 社会价值

- **填补政策落地"最后一公里"**：让农村家庭通过语音对话就能获取精准的资助信息
- **助力数字乡村建设**：帮助农村老年人跨越数字鸿沟，享受数字化服务便利
- **防诈骗守护**：为农村老人提供即时、可靠的防诈骗识别服务
- **可复制推广**：知识库替换后，可推广到其他乡镇、学校，成本低、效果好

## 六、演示方式

### 6.1 演示录屏内容

录屏时长约 4-5 分钟，包含以下 5 幕：

1. **背景引入**：三下乡走访画面，引出"一个家庭，两个需求"的痛点
2. **资助政策通演示**：学生/家长使用场景，展示政策问答 + 条件自测
3. **银龄小保演示**：老人使用场景，展示诈骗识别 + 健康科普 + 养老查询
4. **模式无缝切换**：银龄模式下直接问资助问题，Agent 自动识别切换
5. **总结展望**：项目意义 + 未来扩展方向

### 6.2 交互方式

- 语音输入：点击麦克风按钮，说出问题
- 文字输入：键盘打字输入（备用）
- 语音输出：TTS 合成语音播放，同步显示文字
- 模式选择：首次使用选择身份，后续自动识别切换
